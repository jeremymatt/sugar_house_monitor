Sugar House Monitor API

Available endpoints:

  • POST /sugar_house_monitor/api/ingest_tank.php – upload processed tank readings
  • POST /sugar_house_monitor/api/ingest_pump.php – upload processed pump events
  • POST /sugar_house_monitor/api/ingest_vacuum.php – upload vacuum readings
  • POST /sugar_house_monitor/api/ingest_stacktemp.php – upload stack/ambient thermocouple readings
  • POST /sugar_house_monitor/api/reset.php – truncate server DBs/status (debug only)

Each request must include the shared API key via the X-API-Key header (or an api_key field in the JSON body) and provide the readings/events array in the request body.

Successful ingest will trigger scripts/process_status.py which regenerates the per-component status files (web/data/status_brookside.json, status_roadside.json, and status_pump.json).
